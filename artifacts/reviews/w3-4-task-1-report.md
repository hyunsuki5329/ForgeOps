# W3-4 Task 1 Implementation Report

## Scope

Implemented only Task 1, "Define manifest shape, lifecycle, event range, and deterministic hash," from the approved W3-4 plan.

The change is limited to a closed run-manifest schema, deterministic self-hash helpers, terminal lifecycle and inclusive event-range validation, positive/negative fixtures, and focused tests. It does not implement catalog resolution, cross-run checks, source-reference guards, sensitive-content scanning, storage, encryption, retention, runtime behavior, WBS/RTM updates, Git operations, network access, or external effects.

## Changed resources

- `contracts/forgeops-run-manifest/1.0/schema.json`
  - Defines the closed `forgeops.run-manifest` 1.0 root and nested source-contract, event-stream, artifact, evidence, and decision objects.
  - Preserves the Product Task Contract's opaque `artifact_id` identity domain.
  - Closes run mode, canonical status, artifact kind, classification, evidence tier/type, and decision kind enums.
  - Preserves integer `observed_revision` and the revision/time evidence freshness union.
- `tools/interface_contract/manifest_contract.py`
  - Adds `canonical_manifest_bytes`, `calculate_manifest_sha256`, `validate_lifecycle`, and `validate_manifest`.
  - Excludes only the root `manifest_sha256` field and serializes sorted-key compact JSON as UTF-8 without BOM.
  - Treats `BLOCKED`, `SUCCEEDED`, and `FAILED` as terminal and requires exactly one terminal decision plus `finalized_at`.
  - Rejects non-terminal finalization, invalid timestamp ordering, inclusive event-range mismatches, malformed schemas, and self-hash mismatches with stable errors.
- `fixtures/forgeops-run-manifest/suite.json`
  - Adds primary terminal, non-terminal snapshot, audit replay, counterfactual, revision evidence, time evidence, and blocked terminal manifests.
  - Includes a non-one event range and stored hashes generated by the contract function.
  - Adds lifecycle, range, hash, and closed-root negative cases.
- `tests/interface_contract/test_run_manifest.py`
  - Covers all Task 1 interfaces and fixtures, closed object boundaries, exact enums, terminal lifecycle, integer revisions, opaque source identity, inclusive ranges, malformed schema handling, and UTF-8/root-only canonical hashing.

### Review remediation RED/GREEN

Independent review identified three contract-compatibility issues and two coverage gaps. The new regression run produced two expected failures and two expected errors before remediation:

- `BLOCKED` was not treated as terminal.
- `observed_revision` used a string rather than the required original JSON integer.
- Product Task Contract artifact identities without an `ART-` prefix were rejected.
- An invalid supplied JSON Schema could leak an unstabilized exception.

Separate fixture-first RED runs also demonstrated that the catalog lacked a `BLOCKED` terminal case and a non-one inclusive event range. After the minimal fixes, every targeted test passed independently.
